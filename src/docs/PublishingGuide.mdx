# 퍼블리싱 가이드 (Publishing Guide)

SCENE;er 서비스의 일관된 UI 구현과 유지보수 효율성을 위한 웹 퍼블리싱 및 마크업 규약입니다. 디자이너의 피그마(Figma) 자산이 화면에 정확하고 코드 최적화된 형태로 구현되도록 본 가이드를 준수합니다.

## 1. 기본 마크업 및 시맨틱 태그 구조

화면의 구조는 검색 엔진 최적화(SEO)와 웹 접근성을 고려하여 의미에 맞는 시맨틱 태그로 세분화하여 마크업합니다. id는 사용하지 않고 class만 사용합니다.

### 기본 레이아웃 프레임

- **`scener-app`**: 페이지 전체를 감싸는 최상위 블록. 모든 페이지에서 공통으로 사용합니다.
- **`scener-header`**: 서비스 상단 네비게이션 및 로고 영역. 모든 페이지에서 공통으로 사용합니다.
- **`<main>` 태그의 클래스**: 한 페이지에는 `<main>` 태그가 하나만 존재합니다. 그 페이지가 어떤 화면인지에 따라 클래스 이름을 다르게 짓습니다. 예를 들어 홈 화면의 `<main>`에는 `scener-home`, 아티클 상세 화면의 `<main>`에는 `scener-article`, 마이페이지의 `<main>`에는 `scener-mypage`를 붙입니다. `<main>` 안에는 `<section>`이나 `<div>`로 여러 개의 영역을 나눌 수 있으며, 각 영역은 `scener-{화면이름}-{영역이름}` 형태로 나란히 이름 짓습니다. 아래 예시 코드에서는 어떤 페이지에도 해당하지 않는 임의의 이름인 `scener-sample`을 사용합니다.
- **`scener-footer`**: 하단 푸터 영역. 모든 페이지에서 공통으로 사용합니다.

```html
<div class="scener-app">
  <!-- 페이지 전체를 감싸는 최상위 블록 -->

  <header class="scener-header">
    <div class="scener-logo">
      <a class="scener-logo_area">
        <!-- 로고 클릭 영역(하위 요소이므로 언더스코어) -->
      </a>
    </div>
  </header>

  <main class="scener-sample">
    <!-- <main> 태그의 클래스는 화면 성격에 맞는 이름을 짓습니다 (예시이므로 scener-sample 사용) -->

    <div class="scener-sample-inner">
      <div class="scener-sample-title">
        <!-- 1단계 하위 영역(하이픈) -->
        <span class="scener-sample-title_text"></span>
        <!-- 영역 안의 세부 요소(언더스코어) -->
      </div>

      <section class="scener-sample-visual">
        <!-- 1단계 하위 영역(하이픈) -->
      </section>
    </div>
  </main>

  <footer class="scener-footer">
    <!-- 하단 푸터, 카피라이트 영역 -->
  </footer>
</div>
```

### 네이밍 짓는 순서

클래스 이름은 위에서 아래로 내려가면서, 그 요소가 **어떤 더 큰 것의 일부인지**를 이름에 그대로 누적해서 짓습니다.

1. **블록 이름을 먼저 정한다.** 화면 전체를 대표하는 이름입니다. 위 예시의 `scener-sample`이 여기 해당합니다. 보통 화면 하나당 하나만 존재합니다.
2. **그 블록 안의 큰 영역을 하이픈으로 잇는다.** `scener-sample` 안에 "비주얼 영역"이 있다면 `scener-sample-visual`, "타이틀 영역"이 있다면 `scener-sample-title`처럼 블록 이름 뒤에 하이픈을 붙여 영역 이름을 이어갑니다. 같은 블록 안의 여러 영역은 형제처럼 같은 레벨로 나란히 이름 짓습니다.
3. **그 영역 안의 세부 요소는 언더스코어로 끊는다.** `scener-sample-title` 영역 안에 텍스트가 들어있다면 `scener-sample-title_text`처럼 언더스코어를 붙입니다. 여기서부터는 더 이상 영역을 쌓는 게 아니라, 그 영역을 구성하는 "부품"을 가리키는 것이기 때문에 하이픈이 아닌 언더스코어로 구분합니다.

정리하면, **하이픈은 "영역 안의 또 다른 영역"**을 의미하고, **언더스코어는 "그 영역을 이루는 부품"**을 의미합니다. 이름을 보고 "이게 영역인가, 부품인가"를 구분할 수 있어야 합니다.

## 2. 디자인 토큰 네이밍 규칙

### 토큰 구조 — Primitive → Semantic

모든 디자인 토큰은 2단계 레이어로 나뉩니다.

- **Primitive** (`src/styles/_primitives.scss`): 실제 값(px, hex 등)을 담는 원시 레이어. 의미를 갖지 않으며, **컴포넌트에서 직접 참조하지 않습니다.**
- **Semantic** (`src/styles/_semantic.scss`): primitive를 용도에 맞게 재매핑한 레이어. **컴포넌트는 항상 이 레이어만 참조**합니다.

```scss
// 컴포넌트에서 primitive를 직접 쓰지 않습니다 (지양)
.card {
  background: var(--gray-2);
}

// 항상 semantic 토큰을 거쳐서 씁니다 (권장)
.card {
  background: var(--color-background-elevated);
}
```

### Primitive 레이어

primitive는 의미를 담지 않고, 실제 값을 그대로 이름에 반영합니다.

semantic은 분류, 속성, 변형 순서로 조합합니다. 전체 구조는 다음과 같습니다.

### 재사용 컴포넌트는 다른 규칙을 따릅니다

`scener-button`, `scener-badge`, `scener-event-card`처럼 여러 화면에서 반복해서 쓰이는 컴포넌트는, 특정 화면에 속한 영역이 아니므로 하이픈으로 영역을 쌓는 단계를 거치지 않습니다. 블록 이름 바로 다음에 언더스코어로 부품을 구분합니다.

```html
<div class="scener-event-card">
  <div class="scener-event-card_image"></div>
  <div class="scener-event-card_badges"></div>
  <div class="scener-event-card_overlay"></div>
</div>

<button class="scener-button">
  <span class="scener-button_label">저장하기</span>
</button>
```

### 상태 클래스 — `is-{상태}`

요소의 고정된 종류(category, variant)가 아니라, 인터랙션이나 데이터에 따라 켜지고 꺼지는 **상태**를 표현할 때 사용합니다. 블록/하위요소 클래스 뒤에 공백으로 추가하며, 원래 클래스는 그대로 유지합니다.

```html
<button class="scener-button is-active">저장하기</button>
<button class="scener-button is-disabled" disabled>저장하기</button>
<div class="scener-chip is-active">#아트</div>
```

## 4. 이미지 네이밍 규칙

이미지 파일명은 **형태-의미-상태(또는 순서)** 순서로 조합합니다. 모든 글자는 소문자, 단어 연결은 하이픈(`-`)을 사용합니다. 어떤 슬롯을 채울지는 이미지 종류에 따라 다릅니다. 아이콘은 스타일·컬러가 중요하고, 배너처럼 여러 장 있는 이미지는 순번이 중요합니다.

### 아이콘 — 형태-의미-스타일-컬러

아이콘은 같은 모양이라도 선의 굵기(스타일)와 색이 다른 버전이 여러 개 있을 수 있으므로, 의미 뒤에 스타일과 컬러를 붙입니다.

```
icon-chevron-down-light-black     // 얇은 선(light), 검정
icon-chevron-down-solid-gray      // 채워진 형태(solid), 회색
icon-heart-light-black
icon-heart-solid-lime
```

### 같은 모양, 다른 방향 — 화살표류

화살표나 쉐브론처럼 모양은 같고 방향만 다른 아이콘은, 스타일/컬러 자리에 방향을 넣어 구분합니다.

```
icon-chevron-down-light-black     // 아래 방향
icon-chevron-up-light-black       // 위 방향
icon-chevron-left-light-black     // 왼쪽 방향
icon-chevron-right-light-black    // 오른쪽 방향
```

### 일반 이미지 — 형태-의미-순서

같은 분류의 이미지가 두 개 이상이면, 파일 이름 마지막에 숫자를 추가하여 구분합니다.

```
img-banner-01
img-banner-02
img-poster-exhibition-01
img-poster-exhibition-02
```

### 종합 예시

```
icon-chevron-right-light-black.svg
icon-heart-solid-lime.svg
img-poster-exhibition-01.jpg
logo-symbol.svg
```
