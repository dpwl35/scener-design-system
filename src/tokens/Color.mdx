# Color

SCENE;er의 컬러 시스템은 **Primitive → Semantic** 2단계 토큰 구조를 따릅니다.

- **Primitive**: 실제 색상값을 담는 가장 낮은 레이어. 의미를 갖지 않습니다. (`src/styles/_primitives.scss`)
- **Semantic**: 용도에 맞게 의미를 부여한 레이어. **컴포넌트는 항상 이 레이어만 참조**합니다. (`src/styles/_semantic.scss`)

> 컬러를 바꿔야 할 때는 semantic 레이어의 매핑만 수정하면 됩니다. 컴포넌트 코드는 건드릴 필요가 없습니다.

## Brand

메인 브랜드 컬러입니다. 핵심 CTA 버튼, 로고, 강조 텍스트에만 사용합니다.

<div
  ref={(node) => {
    if (node) {
      node.innerHTML = '';
      node.appendChild(
        createColorGrid({
          columns: 1,
          colors: [
            {
              name: 'Brand Primary',
              variable: '--scener-color-brand-primary',
              textColor: '#1A2E00',
            },
          ],
        }),
      );
    }
  }}
/>

## 사용 규칙

1. 컴포넌트 SCSS에서는 primitive 변수를 직접 쓰지 않고, 항상 `var(--scener-color-...)` semantic 변수를 사용합니다.
2. 카테고리 컬러는 4종 외에 새로운 색을 추가하지 않습니다. 새 장르가 생기면 디자인 시스템 논의를 먼저 거칩니다.
3. 브랜드 컬러(라임)는 한 화면에 과도하게 반복하지 않습니다. 핵심 CTA 1곳에 집중하는 것을 권장합니다.
